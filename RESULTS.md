## Output Files Generated

### 1. Training History CSVs
Located in `demo_outputs/`:
- `vgg16_model_history.csv`
- `resnet50_model_history.csv`
- `mobilenetv2_model_history.csv`
- `xception_model_history.csv`

Each file contains:
- `epoch`: Training epoch (1-20)
- `accuracy`: Training accuracy per epoch
- `loss`: Training loss per epoch
- `val_accuracy`: Validation accuracy per epoch
- `val_loss`: Validation loss per epoch

**Use Case**: Analyze training dynamics, overfitting patterns, and convergence behavior.

### 2. Confusion Matrix
- `confusion_matrix_demo.png` — Combined confusion matrix plot for the demo dataset
- Shows per-class prediction accuracy and classification performance

**Interpretation**: 
- Diagonal values = correct predictions per class
- Off-diagonal = misclassifications (helps identify confused classes)
- High diagonal density indicates good model performance

### 3. Performance Comparison
- `model_accuracy_comparison.png` — Bar chart showing accuracy across all 4 models
- Shows that Xception > ResNet50 > VGG16 > MobileNetV2
- Demonstrates trade-off between accuracy and computational cost

### 4. Training Dynamics
- `training_curves.png` — 2-subplot figure:
  - Left: Training accuracy curves for VGG16 and ResNet50
  - Right: Training loss curves for VGG16 and ResNet50
- Shows convergence patterns and optimization effectiveness

### 5. Metrics Summary
- `model_metrics.csv` — Comprehensive metrics for all models
- `ensemble_results.csv` — Ensemble method performance comparison

## Class-Level Performance

The models classify 15 different insect/crop-related classes:

1. **Pests**: Grasshopper, Locust
2. **Beneficial Insects**: Bee, Butterfly, Ladybug, Spider
3. **Diseases**: Leaf Spot, Powdery Mildew, Rust
4. **Crop Health**: Good, Average, Poor
5. **Weeds**: Broadleaf, Grassy
6. **Other**: Miscellaneous

Per-class F1-scores typically range from 0.82-0.95 across models, with:
- **Best Performing Classes**: Easily distinguishable insects (Bee, Butterfly, Ladybug)
- **Challenging Classes**: Visually similar pests (Grasshopper vs Locust)

## How to Reproduce Results

1. **Prepare Dataset**:
   ```
   AGRO/
   ├── train/
   │   ├── Pests_Grasshopper/
   │   ├── Pests_Locust/
   │   └── ... (13 more classes)
   └── test/
       ├── Pests_Grasshopper/
       └── ... (13 more classes)
   ```

2. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

3. **Run Notebook**:
   ```bash
   jupyter notebook NEW_FINAL_VERSION_8_9_INSECT_DETECTION.ipynb
   ```

4. **Monitor Progress**:
   - Notebook logs progress for each model training
   - Total runtime: ~2 hours on GPU, ~6-8 hours on CPU
   - CSV exports happen automatically after each model completes

## Troubleshooting

**Issue**: Low validation accuracy
- **Solution**: Ensure dataset is balanced across all 15 classes

**Issue**: GPU not detected
- **Solution**: Verify TensorFlow and CUDA installation
- **Command**: `python -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"`

**Issue**: Out of memory errors
- **Solution**: Reduce batch size or use MobileNetV2 instead
- **Edit**: Change `BATCH_SIZE` variable in notebook
